DATABASE ARCHITECTURE

Tables (New):
  1. reports
     - reporter_id (who reported)
     - target_type ('user' | 'room')
     - target_id (UUID)
     - reason (abuse category)
     - status ('pending' | 'resolved' | 'dismissed')
     - RLS: Users see own, admins see all

  2. audit_logs
     - action (role_change, status_change, room_delete)
     - actor_id (who did it)
     - target_id (what changed)
     - metadata (JSON details)
     - RLS: Admin-only read

  3. system_settings
     - key (setting name)
     - value (stored as text)
     - type ('string' | 'number' | 'boolean' | 'json')
     - RLS: Everyone reads, admins write

Helper Functions:
  - log_audit_action() - Automatic action logging
  - get_system_setting() - Retrieve setting value
  - is_setting_enabled() - Boolean check

Triggers:
  - profile_audit_trigger - Logs role/status changes
  - room_delete_audit_trigger - Logs deletions

Performance:
  - Indexes on: status, target, actor, created_at
  - Ready for partitioning (audit_logs growth)
  - Foreign key constraints enforced
